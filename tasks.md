# Tasks

Living checklist surfaced in the web UI. Click a checkbox to toggle. Toggling persists to this file.

The repo ships this file with placeholder examples. Replace them with your real weekly checklist.

## This week

- [ ] /prep <upcoming meeting that matters> the night before
- [ ] /brief tomorrow morning before <first meeting>
- [ ] After <today's important conversation>, /commit the next step and update memory/relationships.md
- [ ] Friday or Saturday, /retro to audit the system

## Context backfill (one-time setup)

These pay off most when filled in early. Treat as a one-time setup checklist when you fork the repo.

- [ ] stakeholders.md: cadence and open thread for each recurring mentor
- [ ] stakeholders.md: cadence and open thread for each active recruiter or founder relationship
- [ ] stakeholders.md: one-line context for each colleague or counterparty you talk to weekly
- [ ] career_thesis.md: name the obvious offer or opportunity you would turn down right now and why
- [ ] priorities.md: top 3-5 active priorities with a one-line "why this matters now"
- [ ] research_arc.md (optional): published work and current threads
- [ ] operating_principles.md: how you decide between speed and care, when you say no, what your weekly rhythm looks like

## Recurring discipline

- [ ] After every meeting that matters, update memory/relationships.md (60 seconds)
- [ ] When priorities shift, update priorities.md the same day
- [ ] Monthly: revisit career_thesis.md
- [ ] Weekly: run /retro
